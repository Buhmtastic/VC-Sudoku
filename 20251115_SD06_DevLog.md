### 2. Building the Game

#### Build Process

**Command Used**:

- `--noconsole`: No console window (GUI-only application)
- `main.py`: Entry point of the application

**Build Output**:

- **Platform**: Windows x86-64
- **Type**: PE32+ GUI executable

### 3. Build Folder Organization

**Initial Issue**: Windows is case-insensitive
- `build/` and `Build/` are the same folder on Windows
- PyInstaller uses `build/` for temporary files
- We want `Build/` for distribution files

**Solution**:
1. Let PyInstaller create `build/` folder
2. Place SudokuMaster.exe in `Build/` folder (same as `build/`)
3. Clean up temporary build files

**Final Structure**:
```
Build/

```

### 4. Git Integration

#### Problem

`.gitignore` was ignoring the `build/` folder:
```
build/    # Ignored by Git
```

#### Solution

Modified `.gitignore` to allow `Build/` folder:
```gitignore
build/      # Ignore lowercase build (temp files)
!Build/     # But include Build folder (distribution)
```

**Git Exception Rule**:
- `!Build/` creates an exception to the `build/` rule
- Build folder and its contents are now tracked by Git

## üìù Lessons Learned

### 1. Case-Sensitivity in Git

**Issue**: Windows is case-insensitive, Git is case-sensitive

**Learning**:
- `build/` and `Build/` are the same folder on Windows
- But Git treats them as different paths
- Can use exception rules (`!Build/`) to handle this

**Best Practice**: Use distinct names for different purposes
- `build/` - temporary build files
- `dist/` - distribution files
- Or use capitalization with Git exceptions

---

### 2. .gitignore Exception Rules

**Pattern**:
```gitignore
folder/        # Ignore folder
!folder/keep/  # But keep this subfolder
```

**Syntax**:
- `!` at start creates exception
- More specific rules override general rules
- Order matters in .gitignore

---

### 3. PyInstaller --onefile vs --onedir

**--onefile**:
- ‚úÖ Single executable file
- ‚úÖ Easy to distribute
- ‚úÖ Clean, simple
- ‚ùå Slower startup (extracts to temp)
- ‚ùå Larger file size

**--onedir**:
- ‚úÖ Faster startup
- ‚úÖ Can include data files easily
- ‚ùå Many files to distribute
- ‚ùå Messy folder structure

**Choice**: --onefile for this project
- Simpler distribution
- Users just download one file
- Startup time acceptable for this game

---

### 4. Executable Size

**15 MB seems large for a simple game**

**Why so large?**:
- Python interpreter: ~5 MB
- Pygame + SDL2: ~8 MB
- Game code + libraries: ~2 MB

**Optimization Options** (not implemented):
- Use UPX compression (--upx-dir)
- Exclude unused modules (--exclude-module)
- Strip debug symbols

**Decision**: 15 MB is acceptable
- Modern storage is cheap
- Download is fast
- No need for extreme optimization
